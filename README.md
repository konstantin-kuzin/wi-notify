# WI Notify

## Назначение

**WI Notify** — расширение для **Google Chrome**, которое показывает **work items Azure DevOps**, назначенные текущему пользователю.


## Загрузка и установка

### 1) Загрузка проекта с GitHub

1. Установите Git, если он ещё не установлен:

2. Откройте терминал в папке, где хотите хранить проект и клонируйте репозиторий:
   ```bash
   git clone https://github.com/konstantin-kuzin/pr-notify.git
   ```


### 2) Установка расширения в Chrome

1. Откройте Chrome и перейдите на страницу:
   ```text
   chrome://extensions
   ```
2. Включите переключатель **«Режим разработчика»** (обычно справа сверху).
3. Нажмите **«Загрузить распакованное расширение»**.
4. Выберите папку проекта `pr-notify` (ту, где лежит `manifest.json`).
5. Убедитесь, что карточка расширения появилась в списке и включена.
6. Закрепите расширение на панели (иконка пазла → pin), чтобы быстро открывать popup.

### 3) Обновление установленной локальной версии

- Чтобы получить новые изменения из GitHub, откройте терминал в папке проекта и выполните:
  ```bash
  git fetch origin
  git pull origin main
  ```

- После обновления кода нажмите **«Обновить»** на `chrome://extensions` (или иконку перезагрузки на карточке расширения), чтобы Chrome перечитал файлы.

## Первоначальная настройка

Откройте страницу настроек: ссылка **«Настройки подключения»** внизу popup или **Параметры расширения** в Chrome.

### 1. Адрес коллекции (корень API)

В поле **«Коллекция»** укажите базовый URL, с которого строятся запросы `_apis/...`:

- **Azure DevOps Server / TFS:** `https://<сервер>/tfs/<имя_коллекции>` (часто совпадает с префиксом URL в браузере при открытии проекта).
- **Облако:** `https://dev.azure.com/<организация>`.

### 2. Проект и репозиторий

- **Проект** — имя team project, как в ADO.
- **Репозиторий (имя или GUID)** — Git-репозиторий, по которому нужен список PR (имя из веб-интерфейса или идентификатор).

### 3. Reviewer-группы

1. В поле поиска введите **полное отображаемое имя группы** (как в ADO), например `Hexa UI Design Review`.
2. Ниже отображаются **уже сохранённые** группы; ненужную можно **удалить** кнопкой **×**.

Если **список групп пуст**, при отборе PR учитываются **все** автоматически определяемые reviewer-группы вашего пользователя **и** PR с **личным** назначением вам. Если в списке есть хотя бы одна группа — в набор ревьюеров входят **только вы** и **эти** группы (подробнее — [docs/documentation.md](docs/documentation.md)).


### Версия API и авторизация

- **`apiVersion`** в интерфейсе **не задаётся**; значение по умолчанию — в [`ado-config.mjs`](ado-config.mjs) (для on-prem часто `6.0-preview`; для `dev.azure.com` при необходимости — `7.1` в коде или вручную в `chrome.storage.local` → `adoConfig`). Ориентируйтесь на текст ошибок в popup.
- **Вход:** по умолчанию **сессия браузера** (куки). 

## Поведение (кратко)

- Автообновление списка **каждые 10 минут** (alarm) и при установке, старте браузера, смене конфига, ручном обновлении из popup.
- **Badge** на иконке: число найденных PR; при ошибке последнего запроса — индикация ошибки, текст ошибки в popup.
- Для каждого попавшего в список PR дополнительно запрашивается **детальный PR** с `includeCommits=true`: в UI попадает **полное описание** и время **последнего коммита на source-ветке** (для «возраста» и просрочки).
- **Сортировка списка:** старые по **обновлению** выше (`updatedAt` — последний source-коммит, иначе дата создания).
- **Просрочка:** если с момента `updatedAt` (коммит или создание) до момента проверки прошло **> 48 ч**, в popup **время** в строке автора выделяется **красным** (без отдельного фона карточки).
- Описание в popup: **раскрываемая панель** под карточкой с **упрощённым markdown** (ссылки, списки, картинки по http(s), базовое форматирование).
- Для описаний с эвристикой «тех ПР» — бейдж **ТЕХ ПР** и кнопка **Approve** (REST `vote: 10` для вашего identity), без автоматизации DOM.
- **Уведомления** о появлении новых id в списке (по сравнению с прошлым успешным обновлением).
- При ошибке API **последний успешный список в storage сохраняется** (поля ошибки и времени проверки обновляются).

## Какие PR попадают в список

- Статус **Active**
— Draft = False
- В `reviewers` есть запись c `id` из набора: **вы** ∪ **группы из сохранённого списка** (или вы ∪ все reviewer-группы пользователя, если список групп в настройках **пуст**)
- Вы не голосовали за это ревью или вас голос сброшен **`vote === 0`**

Подробное описание текущего поведения и полей данных: **[docs/documentation.md](docs/documentation.md)**.

## Устройство проекта

- [`manifest.json`](manifest.json) — MV3, `host_permissions` / `optional_host_permissions`
- [`background.mjs`](background.mjs) — расписание, обновление, badge, уведомления, approve
- [`ado-api.mjs`](ado-api.mjs) — HTTP к ADO, фильтрация, маппинг в модель UI
- [`ado-config.mjs`](ado-config.mjs) — ключ `adoConfig`, значения по умолчанию
- [`options.html`](options.html), [`options.css`](options.css), [`options.mjs`](options.mjs) — настройки
- [`popup.html`](popup.html), [`popup.css`](popup.css), [`popup.mjs`](popup.mjs) — список в popup

## Разрешения

- `alarms`, `storage`, `notifications` — см. манифест.
- Доступ к API — через `host_permissions` / optional для `dev.azure.com`.

## Ограничения

- Один проект и один репозиторий из настроек.
- Другой хост ADO может потребовать правки `host_permissions` в манифесте.
- Markdown в popup — упрощённый, не копирует рендер ADO целиком.

## Разработка

```bash
node --check background.mjs
node --check ado-api.mjs
node --check ado-config.mjs
node --check options.mjs
node --check popup.mjs
```
